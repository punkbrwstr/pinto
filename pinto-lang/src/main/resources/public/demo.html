Pinto Programming Language

What is Pinto?

Pinto is a domain-specific programming language for working with time series data. Pinto expressions produce one thing: tables of data with rows that correspond to periodic points in time. Each table column contains a consistent type of data, but the values could be constant or time-varying. Columns also have a text header for identifying metadata. Pinto expressions can be evaluated to produce values for any periodicity over any range of time. As a concatenative language that uses postfix notation, Pinto expressions are comprised of a sequence of functions that operate on the the same table in left-to-right order. The table is set up as a stack, with new columns added to the right and functions operating on the rightmost column first. Pinto indexing expressions allow convenient filtering of the table--passing only selected columns to the next function by referencing a column's header value or position in the table.

For language details see the Pinto Language Reference.

Key features

  • Expressive: Concise code using recognizable symbols and efficient postfix notation
  • Batteries included: Integrated Bloomberg or IEX data, charting, rich library of statistical functions
  • Efficient: Lazy evaluation, range-based caching for nullary functions
  • Extensible: User-defined functions behave just like primitives
  • Interoperable: Accessible through an http interface (works great with Python or SAS)

Why Pinto?

Pinto was designed to prepare financial market data for use in models or visualizations. A Pinto one-liner can perform data transformations that would require many lines of Pandas operations, multiple SAS data steps, or an Excel sheet full of formulas. With Pinto's side-effect-free functional paradigm, the expression will return the same values every time. And, Pinto can reevaluate the same expression for a different frequency or an expand the range of dates without changes to the code.

Pinto combines the efficiency of an HP-12c calculator, the simplicity of Postscript, the tersity of J, and the familiarity of Python indexing.

Pinto by example


        # Here are examples of Pinto code that you can run in the interpreter window above 
        # Historical market data is provided by IEX and available for US stocks

        # First let's define a function that returns a date range
        # B is a periodicity literal for business days
        # We will evaluate our expressions over this range
        :my_range [] B 2018-03-01 2018-03-07 def
        
        # We can retrieve stock prices using a market data literal expression
        # and evaluating it over our date range
        $cmg,loco,yum:CLOSE$ my_range eval
        
        # We can see which columns the eval function expects from the stack
        # by using the help function
        :eval help
        
        # Next, define a function to return the stock prices with labels
        :my_portfolio $cmg,loco,yum:CLOSE$ {Chipotle,El Pollo,Yum} def
        
        # Using a the rolling function we can compute a moving averages for our portfolio
        my_portfolio ([:&] {size: 20} rolling mean) my_range eval
        
        # This will compute the difference between two moving averages
        my_portfolio ([:&] {size: 20} rolling mean) ([:3] {size: 200} rolling mean) {n:3} - my_range eval
